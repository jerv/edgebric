## mKB API Format (source-confirmed from JS microservice)

### Create Dataset
```
POST /api/mkb/v1/datasets
Content-Type: application/json
{"datasetName": "my-dataset", "model": "nomic-embed-text"}
```

### Upload Chunks
**NOT JSON. NOT a JSON array. NDJSON in a multipart form field named "chunks".**

```
POST /api/mkb/v1/datasets/{name}/chunks
Content-Type: multipart/form-data; boundary=...
Form field: "chunks" (filename: "chunks.mdf")
```

File content (NDJSON lines joined by `\n\n`):
```
{"chunkTotal":2,"searchPrefix":null}

{"chunk":"text of chunk 1","embedding":[0.1, 0.2, ...]}

{"chunk":"text of chunk 2","embedding":[0.3, 0.4, ...]}
```

- chunkId is auto-generated by mKB as `{datasetName}-{0-indexed-position}`
- Embeddings must be pre-computed via mILM before upload
- Response is SSE streaming progress events

### Search
```
POST /api/mkb/v1/search
Content-Type: application/json
{"datasetName": "my-dataset", "prompt": "query text", "topN": 5}
```

mKB embeds the prompt internally via `GEN_AI_EMBEDDING_URI` → mILM.
No need to pre-compute the query embedding.

## mKB Configuration (in deploy-mimik.sh)

```json
{
  "env": {
    "MCM.DB_EMBEDDING_SUPPORT": "true",
    "GEN_AI_EMBEDDING_URI": "http://localhost:8083/api/mim/v1/embeddings",
    "GEN_AI_API_KEY": "<same as MILM_API_KEY>"
  }
}
```

`MCM.DB_EMBEDDING_SUPPORT: true` tells mKB to handle embeddings internally.
`GEN_AI_EMBEDDING_URI` points to mILM's embeddings endpoint.
